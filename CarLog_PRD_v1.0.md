## 1. 문서 개요

### 1.1 목적

본 문서는 법인차량 관리 시스템(CarLog)의 제품 요구사항 및 기능 정의를 기술한다. 이 시스템은 여러 대의 법인차량을 운행하는 기사들이 운행·주유·정비·지출 기록을 모바일/웹 환경에서 손쉽게 입력하고, 관리자가 이를 실시간으로 집계하여 국세청 업무용 차량 운행기록부를 자동 생성·제출할 수 있도록 지원한다.

### 1.2 범위

#### In-Scope (현 Phase 포함)

- 기사 로그인 (차량 선택 + 이름 입력)
- 4가지 기록 입력: 운행 / 주유 / 정비 / 지출
- 기사 개인 대시보드 (이달 통계, 최근 기록)
- 관리자 차량 현황 집계 및 전체 기록 조회
- 기간·차량별 필터링
- CSV 다운로드 (국세청 양식)
- PDF 출력 (인쇄·저장)
- 이메일 공유

#### Out-of-Scope (현 Phase 제외)

- GPS 자동 위치 추적
- 실시간 차량 모니터링
- 사고 보고 / 보험 청구 워크플로우
- 회계 시스템 연동 (ERP)
- 카카오톡 API 직접 연동 (현재 파일 수동 공유)
- 다국어 지원

### 1.3 용어 정의

| 용어 | 정의 |
|------|------|
| 법인차량 | 회사 소유 업무용 차량 |
| 운행기록부 | 국세청 제출용 업무용 차량 운행 기록 장부 |
| 기사 | 법인차량을 운행하는 담당 직원 |
| 관리자 | 전체 차량 현황을 조회·집계하는 관리 담당자 |
| 운행 기록 | 출발지, 도착지, 시간, 계기판 수치 등 운행 내역 |
| 주유 기록 | 주유소, 주유량, 금액, 결제수단 등 주유 내역 |
| 정비 기록 | 정비 유형, 업체, 금액 등 차량 정비 내역 |
| 지출 기록 | 주차비, 통행료, 세차비 등 기타 차량 관련 지출 |
| CSV | Comma-Separated Values, 엑셀 호환 데이터 파일 |
| PDF | 국세청 제출용 인쇄 문서 포맷 |
| PIN | Personal Identification Number, 개인 식별 번호 |

## 2. 제품 개요

### 2.1 제품 비전

> 법인차량 운행 기록을 디지털화하여, 기사는 간편하게 입력하고 관리자는 국세청 제출 자료를 자동으로 생성한다.

### 2.2 타겟 사용자 / 페르소나

#### 페르소나 A — 기사 (기록 입력자)

| 항목 | 내용 |
|------|------|
| 이름 | 김철수 (40대, 법인차량 전담 기사) |
| 목표 | 운행 후 빠르게 기록 입력 → 서류 작업 최소화 |
| 불편사항 | 수기 운행일지 작성 번거로움, 정산 시 누락 발생 |
| 사용 환경 | 스마트폰 중심, 간단한 UI 선호 |

#### 페르소나 B — 관리자 (집계·제출 담당자)

| 항목 | 내용 |
|------|------|
| 이름 | 이수진 (30대, 총무/경영지원 담당) |
| 목표 | 월말 국세청 제출 자료 자동 생성, 수기 취합 업무 제거 |
| 불편사항 | 기사별 엑셀 파일 수동 취합, 누락·오류 빈번 |
| 사용 환경 | PC 및 태블릿, 엑셀·PDF 문서 작성 익숙 |

### 2.3 사용자 문제 정의

| 대상 | 현재 문제 | 해결 방향 |
|------|-----------|-----------|
| 기사 | 수기 운행일지 작성 → 분실·누락 | 모바일 앱에서 실시간 입력 |
| 기사 | 주유·정비 영수증 보관 어려움 | 앱에서 즉시 디지털 기록 |
| 관리자 | 기사별 파일 수동 수집·취합 | 관리자 화면에서 자동 집계 |
| 관리자 | 국세청 양식 수작업 변환 | CSV·PDF 자동 생성·다운로드 |

## 4. 기능 상세 정의서

### F-01. 기사 로그인

#### F-01.1 개요

기사는 차량 목록에서 담당 차량을 선택하고 본인 이름을 입력하여 로그인한다. 차량별로 등록된 기사 명단과 입력 이름이 일치해야 접근이 허용된다.

#### F-01.2 기능 상세

| 항목 | 내용 |
|------|------|
| 입력 | 차량 선택 (그리드), 기사 이름 텍스트 입력 |
| 검증 | 선택 차량에 등록된 기사 명단과 입력 이름 일치 여부 확인 |
| 출력 | 대시보드 화면 이동, 세션 유지 (새로고침 시 재로그인) |
| 차량 등록 | 시스템 관리자가 사전에 차량번호 및 기사 명단 등록 |
| 보안 | PIN 미사용, 이름 일치 방식 (MVP 간소화) |

#### F-01.3 사용자 시나리오

**Happy Path (정상 흐름)**

1. 차량 그리드에서 담당 차량 선택
2. 이름 입력란에 본인 이름 입력
3. 로그인 버튼 클릭
4. 대시보드 화면으로 이동

**Edge Cases (예외 처리)**

- 차량 미선택 시 → '차량을 선택하세요' 토스트 메시지
- 이름 미입력 시 → '이름을 입력하세요' 오류 표시
- 등록되지 않은 이름 입력 시 → 해당 기사가 아님 안내
- 관리자 페이지 접근 시 → 별도 관리자 코드 입력

#### F-01.4 비기능 요구사항

- 로그인 처리 시간 1초 이내
- 오프라인 환경에서도 로컬 저장 데이터 접근 가능

#### F-01.5 제외 범위

- 소셜 로그인, OTP 인증 (Phase 2 고려)
- 생체 인증 (Phase 3 고려)

---

### F-03. 운행 기록 입력

#### F-03.1 개요

기사는 운행 후 날짜, 출발지, 도착지, 출발/도착 시간, 출발/도착 계기판 수치, 운행 목적을 입력한다. 운행 거리는 계기판 수치 차이로 자동 산출된다.

#### F-03.2 기능 상세

| 항목 | 내용 |
|------|------|
| 필수 입력 | 날짜, 출발지, 도착지, 출발 계기(km), 도착 계기(km), 운행 목적 |
| 선택 입력 | 출발 시간, 도착 시간, 비고 |
| 운행 목적 | 업무 운행 / 출퇴근 / 거래처 방문 / 배송 / 기타 (드롭다운) |
| 자동 계산 | 운행 거리 = 도착 계기 - 출발 계기 (저장 시 자동 산출) |
| 저장 | 클라우드 스토리지에 즉시 저장, 기사 정보 자동 포함 |
| 국세청 매핑 | 업무용 차량 운행기록부 필드와 1:1 매핑 |

#### F-03.3 사용자 시나리오

**Happy Path (정상 흐름)**

1. 운행 기록 메뉴 진입
2. 날짜 자동 설정 (오늘)
3. 출발지·도착지·계기판 수치 입력
4. 운행 목적 선택
5. 저장 버튼 클릭
6. 토스트 메시지 확인 후 대시보드 복귀

**Edge Cases (예외 처리)**

- 도착 계기 < 출발 계기 → '계기판 수치를 확인하세요' 경고
- 날짜 미입력 → 오늘 날짜 자동 입력
- 운행 목적 미선택 → 저장 시 경고
- 저장 실패 시 → 재시도 안내

#### F-03.4 비기능 요구사항

- 저장 처리 2초 이내
- 오프라인 임시 저장 후 온라인 복구 시 동기화 (Phase 2)

#### F-03.5 제외 범위

- GPS 자동 경로 기록 (Phase 3)
- 사진 첨부 (Phase 2)

---

### F-04. 주유 기록 입력

#### F-04.1 개요

기사는 주유 후 날짜, 주유소명, 주유량, 금액, 연료 종류, 결제 방법을 입력한다. 현재 계기판 수치도 함께 기록하여 연비 추적이 가능하다.

#### F-04.2 기능 상세

| 항목 | 내용 |
|------|------|
| 필수 입력 | 날짜, 주유량(L), 금액(원), 연료 종류 |
| 선택 입력 | 주유소명, 현재 계기(km), 결제 방법 |
| 연료 종류 | 휘발유 / 경유 / LPG / 전기 (드롭다운) |
| 결제 방법 | 법인카드 / 현금 / 개인카드 (드롭다운) |
| 저장 | 클라우드 스토리지에 즉시 저장 |

#### F-04.3 사용자 시나리오

**Happy Path (정상 흐름)**

1. 주유 기록 메뉴 진입
2. 날짜 자동 설정 (오늘)
3. 주유소, 주유량, 금액 입력
4. 연료 종류 및 결제 방법 선택
5. 저장 버튼 클릭
6. 대시보드 복귀

**Edge Cases (예외 처리)**

- 주유량 0 이하 입력 → 유효성 검증 경고
- 금액 미입력 → 저장 시 경고
- 저장 실패 시 → 재시도 안내

#### F-04.4 비기능 요구사항

- 저장 처리 2초 이내

#### F-04.5 제외 범위

- 영수증 사진 첨부 (Phase 2)
- 주유소 위치 자동 감지 (Phase 3)

---

### F-05. 정비 기록 입력

#### F-05.1 개요

기사는 차량 정비 후 정비 유형, 정비 업체, 금액, 계기판 수치, 결제 방법을 입력한다. 정기점검부터 사고수리까지 다양한 정비 유형을 드롭다운으로 선택할 수 있다.

#### F-05.2 기능 상세

| 항목 | 내용 |
|------|------|
| 필수 입력 | 날짜, 정비 유형, 금액(원) |
| 선택 입력 | 정비 업체명, 현재 계기(km), 결제 방법, 비고 |
| 정비 유형 | 엔진오일 교환 / 타이어 교환 / 브레이크 정비 / 에어필터 교환 / 배터리 교환 / 정기 점검 / 사고 수리 / 기타 |
| 결제 방법 | 법인카드 / 현금 / 개인카드 |

#### F-05.3 사용자 시나리오

**Happy Path (정상 흐름)**

1. 정비 기록 메뉴 진입
2. 정비 유형 선택
3. 업체명, 금액 입력
4. 결제 방법 선택
5. 저장 버튼 클릭
6. 대시보드 복귀

**Edge Cases (예외 처리)**

- 정비 유형 미선택 → 저장 시 경고
- 금액 0 이하 → 유효성 검증 경고

#### F-05.4 비기능 요구사항

- 저장 처리 2초 이내

#### F-05.5 제외 범위

- 정비 예약 알림 (Phase 3)
- 정비 이력 기반 다음 점검 일정 자동 제안 (Phase 3)

---

### F-06. 지출 기록 입력

#### F-06.1 개요

주차비, 통행료, 세차비 등 차량 운행에 수반되는 기타 지출을 기록한다.

#### F-06.2 기능 상세

| 항목 | 내용 |
|------|------|
| 필수 입력 | 날짜, 지출 유형, 금액(원) |
| 선택 입력 | 결제 방법, 비고 |
| 지출 유형 | 주차비 / 통행료 / 세차비 / 보험료 / 차량 검사비 / 기타 |
| 결제 방법 | 법인카드 / 현금 / 개인카드 |

#### F-06.3 사용자 시나리오

**Happy Path (정상 흐름)**

### F-10. CSV 다운로드 (국세청 양식)

#### F-10.1 개요

관리자는 기간과 차량을 선택하여 국세청 업무용 차량 운행기록부 CSV 파일을 생성·다운로드한다. CSV는 운행·주유·정비·지출 기록을 통합하며 UTF-8 BOM 인코딩으로 엑셀 호환을 보장한다.

#### F-10.2 기능 상세

| 항목 | 내용 |
|------|------|
| 필터 조건 | 기간 (연·월 선택), 차량 (전체 또는 특정 차량) |
| 파일명 | `법인차량운행기록_YYYY-MM.csv` |
| 인코딩 | UTF-8 BOM (엑셀 한글 호환) |
| 컬럼 구성 | 차량번호, 기사명, 날짜, 구분, 출발지, 도착지, 출발시간, 도착시간, 출발계기, 도착계기, 운행거리, 운행목적, 주유량, 주유금액, 주유소, 연료, 정비유형, 정비업체, 정비금액, 지출유형, 지출금액, 결제방법, 비고 |
| 정렬 | 차량번호 → 날짜 오름차순 |
| 국세청 대응 | 소득세법 시행령 제78조의3 업무용 차량 운행기록부 기준 |

#### F-10.3 사용자 시나리오

**Happy Path (정상 흐름)**

1. 다운로드 탭 진입
2. 기간 및 차량 선택
3. CSV 다운로드 버튼 클릭
4. 브라우저 다운로드 팝업 확인
5. 로컬 저장 완료

**Edge Cases (예외 처리)**

- 해당 기간에 기록 없음 → 헤더만 있는 빈 CSV 다운로드
- 전체 차량 선택 시 모든 차량 데이터 통합 출력
- 특수문자 포함 데이터 → 쌍따옴표로 이스케이프 처리

#### F-10.4 비기능 요구사항

- 파일 생성 시간 3초 이내 (1년치 전체 데이터 기준)
- 100개 차량 × 12개월 데이터 처리 가능

#### F-10.5 제외 범위

- 서버 사이드 생성 (현재 클라이언트 생성)
- 디지털 서명·전자제출 연동 (Phase 3)

### F-11. PDF 출력

#### F-11.1 개요

관리자는 기간·차량을 선택하여 국세청 제출용 PDF 보고서를 생성한다. 차량별 요약(운행거리·주유·정비·지출 합계)과 상세 기록 테이블이 포함된 공식 문서 형태로 출력된다.

#### F-11.2 기능 상세

| 항목 | 내용 |
|------|------|
| 필터 조건 | 기간 (연·월), 차량 (전체 또는 특정 차량) |
| 문서 구성 | 표지 정보 → 차량별 요약 → 운행 기록 테이블 → 주유/정비/지출 테이블 |
| 요약 섹션 | 총 운행거리, 주유 합계, 정비 합계, 지출 합계, 총 지출 |
| 출력 방식 | 브라우저 인쇄 다이얼로그 호출 → PDF 저장 또는 직접 인쇄 |
| 폰트 | 맑은 고딕 (한글 호환), 11px 기본 |
| 페이지 | A4, 자동 페이지 분기 |

#### F-11.3 사용자 시나리오

**Happy Path (정상 흐름)**

1. PDF 출력 버튼 클릭
2. 새 탭에서 HTML 보고서 생성
3. 브라우저 인쇄 다이얼로그 자동 호출
4. PDF 저장 또는 인쇄기 선택

**Edge Cases (예외 처리)**

- 팝업 차단 시 → '팝업을 허용해 주세요' 안내
- 빈 데이터 → '해당 기간에 기록이 없습니다' 표시
- 대량 데이터 → 자동 페이지 분기

#### F-11.4 비기능 요구사항

- 보고서 렌더링 2초 이내
- 인쇄 다이얼로그 호출 3초 이내

#### F-11.5 제외 범위

- 서버 사이드 PDF 생성 (현재 브라우저 인쇄)
- 전자 서명 삽입 (Phase 3)

### 6.1 기술 스택 (MVP)

## 8. 리스크 및 가정 사항

### 8.1 가정 사항

- 사용자는 스마트폰 또는 PC 웹 브라우저를 통해 앱에 접근한다
- 차량 정보 및 기사 명단은 시스템 배포 전 코드에 사전 등록된다
- 국세청 업무용 차량 운행기록부 양식은 소득세법 시행령 제78조의3 기준을 따른다
- 인터넷 연결이 가능한 환경에서 주로 사용된다 (오프라인 지원은 Phase 2)
- 관리자 코드는 운영 전 내부 공유되며, 별도 계정 관리 시스템이 없다

### 8.2 리스크 매트릭스

| 리스크 | 발생 가능성 | 영향도 | 대응 방안 |
|--------|-------------|--------|-----------|
| 기사 이름 오입력으로 로그인 실패 | 중 | 중 | 이름 힌트 표시 또는 차량별 기사 목록 노출 (Phase 2 PIN 도입) |
| Cloud Storage 데이터 유실 | 저 | 상 | 정기적 CSV 백업 권고, 로컬 임시 저장 로직 추가 |
| 국세청 양식 변경 | 저 | 상 | 양식 버전 관리 체계 도입, 분기별 검토 |
| 브라우저 팝업 차단으로 PDF 출력 실패 | 중 | 중 | 팝업 허용 안내 가이드 제공, 대체 HTML 출력 방식 제공 |
| 다수 기사 동시 접속 시 데이터 충돌 | 중 | 상 | Last-Write-Wins 방식 적용, 저장 타임스탬프 기록 |
